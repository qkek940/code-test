# MSDS 양식 프롬프트 빌더 — 사용법

업로드한 MSDS(PDF)를 읽어 입력 항목을 자동으로 채우고, Claude Design용
변환 프롬프트를 생성·복사하는 단일 페이지 도구입니다.

## 실행 방법 (중요)

> ⚠️ **`index.html`을 더블클릭(`file://`)으로 열면 AI 추출이 동작하지 않습니다.**
> 브라우저 보안정책상 `file://` 페이지는 외부 API(Gemini) 호출이 차단됩니다.
> 반드시 **로컬 서버**로 여세요.

### 가장 쉬운 방법 — 실행 스크립트 더블클릭
- **macOS**: `run-mac.command` 더블클릭
- **Windows**: `run-windows.bat` 더블클릭
- **Linux**: 터미널에서 `./run-linux.sh`

→ 로컬 서버가 뜨고 브라우저에서 `http://localhost:8000/index.html` 이 자동으로 열립니다.
(Python 3 필요 — 대부분의 Mac/Linux에 기본 설치. Windows는 python.org에서 설치)

### 또는 터미널에서 직접
```bash
python3 -m http.server 8000
# 브라우저에서 http://localhost:8000/index.html 접속
```

## API 키 설정 (AI 자동추출용)

두 가지 방법 중 하나를 쓰세요. 둘 다 있으면 `config.js` 가 우선합니다.

### 방법 1 — index.html에 직접 내장 (단독 파일)
`index.html` 상단의 다음 줄에 본인 키를 넣으면 파일 하나로 동작합니다.
```js
const EMBEDDED_GEMINI_KEY = "여기에-키";
```
⚠️ 이 경우 키가 `index.html` 안에 들어가므로, **공개 GitHub에 올리면 키가 노출**됩니다.
공개 저장소에 올릴 거면 방법 2를 쓰거나 저장소를 private으로 두세요.

### 방법 2 — config.js 분리 (공개저장소 안전)
1. `config.example.js` 를 복사해 `config.js` 로 저장
2. `config.js` 에 본인 키 입력 → `window.GEMINI_API_KEY = "여기에-키";`
3. `config.js` 는 `.gitignore` 되어 GitHub에 올라가지 않습니다. 배포 시 `index.html` 과 **같은 폴더**에 함께 두세요.

> 🔒 키는 클라이언트에서 직접 호출되므로, 노출 시 악용을 막으려면 Google Cloud
> 콘솔에서 키에 **HTTP 리퍼러 제한**(본인 도메인)과 **API 제한**(Generative
> Language API)을 거는 것을 권장합니다.

## 사용 흐름

1. 상단에서 양식 선택 (작업공정별 관리요령 / 물질경고표지)
2. MSDS **PDF** 업로드 *(자동추출은 PDF만 지원 — xlsx·docx·pptx는 PDF로 변환 후 업로드)*
3. **데이터 추출 시작** → PDF를 분석해 빈 항목을 자동으로 채움
   (직접 입력한 값은 보존되고, 빈 칸만 채워집니다)
4. 내용 검토·수정
5. **인쇄** — 미리보기 패널 우측 상단 **인쇄** 버튼 → 선택한 양식 시트만 A4 한 장으로 출력
   (브라우저 인쇄 대화상자에서 "PDF로 저장"을 고르면 PDF 파일로도 저장됩니다.)
6. (선택) **프롬프트 복사** → Claude Design 대화창에 붙여넣고 같은 MSDS 파일 첨부

## Firestore 저장 (선택)

`데이터 추출 시작` 시 입력값·생성 프롬프트가 Firestore `submissions` 컬렉션에
저장됩니다. 동작하려면 `firestore.rules` 의 보안 규칙을 배포해야 합니다:
```bash
firebase deploy --only firestore:rules
```
